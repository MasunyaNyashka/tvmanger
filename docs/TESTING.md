# Тестирование

Этот раздел описывает тестовую стратегию проекта и указывает, где лежат детальные инструкции по каждому типу тестов.

## Стратегия тестирования

Проект использует многоуровневый подход:

- Unit-тесты: проверка бизнес-логики сервисов в изоляции (моки зависимостей).
- Интеграционные тесты: проверка связки Spring + БД (в тестах используется Testcontainers).
- Functional API тесты: проверка сквозных HTTP-сценариев на поднятом тестовом стенде.
- Нагрузочные тесты: проверка устойчивости и времени ответа под параллельной нагрузкой.

Приоритет запуска в CI/локально:

1. Unit-тесты (быстро и дешево по времени).
2. Интеграционные тесты (контракт с БД и конфигурацией).
3. Functional API тесты (бизнес-потоки между сервисами).
4. Нагрузочные тесты (по расписанию или перед релизом).

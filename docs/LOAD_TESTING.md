# Нагрузочное тестирование

## Назначение

Нагрузочные тесты проверяют устойчивость API под параллельной нагрузкой и помогают оценить:

- долю неуспешных запросов;
- время ответа (`p95`, `p99`);
- стабильность бизнес-потоков под нагрузкой.

## Инструмент

- `k6`
- сценарий: `load-tests/api-load.js`

## Предусловия

1. Поднять тестовый стенд:

```bash
docker compose -f docker-compose.test.yml up --build
```

2. Установить `k6` (пример для Windows):

```powershell
winget install k6 --source winget
```

## Что делает текущий сценарий

Один виртуальный пользователь выполняет цепочку:

1. регистрация клиента;
2. логин клиента;
3. получение списка тарифов;
4. создание заявки на подключение;
5. создание сервисной заявки.

## Профиль нагрузки

Текущий профиль `ramping-vus`:

- старт с `1` VU;
- рост до `5` VU за `30s`;
- рост до `10` VU за `1m`;
- снижение до `0` VU за `30s`.

## Пороговые значения

- `http_req_failed < 10%`
- `http_req_duration p(95) < 1500ms`
- `http_req_duration p(99) < 3000ms`
- `successful_flows > 80%`

## Запуск

Базовый запуск:

```bash
k6 run load-tests/api-load.js
```

С явными адресами сервисов:

```bash
k6 run ^
  -e AUTH_BASE_URL=http://localhost:18081 ^
  -e TARIFF_BASE_URL=http://localhost:18082 ^
  -e ORDER_BASE_URL=http://localhost:18083 ^
  -e SERVICE_REQUEST_BASE_URL=http://localhost:18084 ^
  load-tests/api-load.js
```

## Что фиксировать по результатам

- дата/время и версия стенда;
- профиль нагрузки (VU, длительность, сценарий);
- итоговые `p95/p99`;
- `%` ошибок HTTP и бизнес-ошибок;
- узкие места и план оптимизации.

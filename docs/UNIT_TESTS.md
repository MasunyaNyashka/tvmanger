# Unit-тесты

## Назначение

Unit-тесты проверяют бизнес-логику на уровне сервисных классов без запуска полного Spring-контекста и без реальной БД.

## Библиотеки

Unit-тесты в проекте используют:

- JUnit 5 (`org.junit.jupiter`) для запуска тестов.
- Mockito (`org.mockito`, через `spring-boot-starter-test`) для моков и проверки взаимодействий.
- AssertJ (`org.assertj`) для выразительных проверок.
- Spring Boot Test starter (`spring-boot-starter-test`) как базовый набор test-зависимостей.

## Описание тестовых сценариев

### Auth Service

Файл: `auth-service/src/test/java/com/masunya/auth/service/AuthServiceTest.java`

- Регистрация нового клиента: создание пользователя, хеширование пароля, выдача JWT.
- Отказ в регистрации при дублировании username.
- Логин клиента: успешная выдача токена без admin-аудита.
- Логин администратора: запись события `ADMIN_LOGIN` в audit log.
- Ошибки аутентификации: неизвестный пользователь и неверный пароль.

Файл: `auth-service/src/test/java/com/masunya/auth/security/JwtUtilTest.java`

  - Генерация JWT с корректными claim-полями (`sub`, `username`, `role`, `iat`, `exp`).

### Tariff Service

Файл: `tariff-service/src/test/java/com/masunya/tariff/service/TariffServiceTest.java`

- Получение публичного списка и тарифа по id (только неархивные).
- Ошибка `Tariff not found` для отсутствующего тарифа.
- Создание тарифа: нормализация данных, сохранение, запись аудита.
- Ошибка при дублировании имени тарифа.
- Обновление тарифа: изменение полей, аудит изменений.
- Архивация/разархивация тарифа и корректный audit action.
- Удаление тарифа и обработка случая отсутствующего тарифа.

### Order Service

Файл: `order-service/src/test/java/com/masunya/order/service/ConnectionOrderServiceTest.java`

- Создание заявки на подключение: trim входных данных, статус `SUBMITTED`.
- Чтение собственной заявки и ошибка при отсутствии.
- Админ-фильтрация заявок по статусу и диапазону дат.
- Обновление статуса заявки:
  - валидный переход,
  - создание клиентского тарифа при активации,
  - аудит смены статуса.
- Ошибки:
  - невалидный переход статусов,
  - отсутствующая заявка.

Файл: `order-service/src/test/java/com/masunya/order/service/ClientTariffServiceTest.java`

- Назначение клиентского тарифа администратором и audit log.
- Получение тарифов клиентом и админом (с фильтром и без).
- Создание клиентского тарифа из активированной заявки (без audit log).
- Изменение тарифа клиента с записью деталей в аудит.
- Ошибка при отсутствии клиентского тарифа.

### Service Request Service

Файл: `service-request-service/src/test/java/com/masunya/servicerequest/service/ServiceRequestServiceTest.java`

- Создание сервисной заявки: trim полей, статус `SUBMITTED`.
- Чтение собственной заявки и ошибка при отсутствии.
- Админ-выборка заявок по статусу, типу и диапазону дат.
- Обновление статуса сервисной заявки:
  - валидный переход и аудит,
  - допустимый переход в тот же статус,
  - ошибка для невалидного перехода.
- Ошибка при попытке обновить несуществующую заявку.

## Запуск

Из корня проекта:

```bash
mvn test
```

Только unit-тесты конкретного модуля (пример):

```bash
mvn -pl auth-service test
```
